🧠 Claude용 프롬프트 — “이 언리얼 게임 프로젝트의 모든 브랜치를 심층 분석하라”

명령:
아래 깃허브 저장소의 모든 브랜치를 완전히 분석하라.

GitHub 링크: [여기에 링크 입력]

🎯 분석 목표
1️⃣ 브랜치 구조 및 목적 파악

존재하는 모든 브랜치 이름을 나열하고, 각 브랜치의 역할/목적(예: 기능 개발, 버그 수정, 실험, 릴리스 등)을 추론하라.

각 브랜치가 main, master, develop 등과 어떤 관계를 가지는지 (파생 여부, 머지 상태, 분기 시점 등)을 설명하라.

브랜치 구조를 시각적 트리 형태로 서술해라. (예: main → develop → feature/...)

2️⃣ 코드 및 언리얼 프로젝트 구조 분석

브랜치별 언리얼 프로젝트 구조(Source/, Content/, .uproject 등)를 중심으로 분석하라.

주요 모듈 구조(GameModule, EditorModule, Runtime 등) 및 클래스 계층을 파악하라.

사용된 언리얼 시스템 및 기능(예: Gameplay Ability System, AI Controller, Behavior Tree, DataAsset, Blueprints, C++ 연동 등)을 식별하고 설명하라.

코드 네이밍 규칙, 아키텍처 패턴(MVC, ECS, Gameplay Framework 등) 사용 여부를 평가하라.

C++과 블루프린트의 혼용 비율과 그 이유를 추론하라.

3️⃣ 개발 흐름 및 협업 패턴 분석

커밋 메시지의 규칙성, 구체성, 일관성을 분석하라.

Pull Request가 있다면 코드 리뷰, 승인 과정, 협업 스타일을 설명하라.

여러 개발자가 기여했다면, 커밋 기록을 통해 기능 단위 분업 구조를 추정하라 (예: UI 담당, AI 담당, 시스템 담당 등).

4️⃣ 진행 상태 및 프로젝트 완성도 평가

브랜치별 기능 구현 진행도를 평가하라 (예: 완료 / 진행 중 / 미완성 / 폐기).

프로젝트 전체가 현재 어떤 단계에 있는지 추론하라 (프로토타입 / 알파 / 베타 / 출시 / 유지보수 등).

각 브랜치의 코드 안정성과 빌드 가능성(에러, 충돌 가능성 등)을 추정하라.

5️⃣ 문제점 및 개선 제안

브랜치 관리, 코드 구조, 모듈 구성, 네이밍, 의존성 관리 등에서의 문제점을 구체적으로 지적하라.

더 효율적인 브랜치 전략(예: Git Flow, Trunk-based, Release branching 등)을 제안하라.

언리얼 개발 관점에서의 개선 제안(예: UObject 관리 최적화, Blueprint 정리, 메모리 관리, 모듈 분리 등)을 포함하라.

리팩토링, 테스트, 문서화 관련 구체적 권장 사항을 제시하라.

📋 1. 브랜치 요약표
| 브랜치명 | 역할/목적 | 상태 | 주요 시스템 | 메인 병합 여부 |
|-----------|------------|--------|------------------|----------------|
| main      | 출시용 안정 브랜치 | 안정 | Core Gameplay, UI | O |

🧾 2. 전체 분석 요약
[프로젝트 개요]
- 전체 브랜치 수: N개  
- 개발 패턴: Git Flow 기반  
- 주요 시스템: Gameplay Ability System, UI Framework, Behavior Tree  
- C++/Blueprint 비율: 약 70/30  
- 코드 품질: 상 (클래스 구조 명확, 모듈 분리 우수)  

[협업 패턴]
- 커밋 메시지 일관성: 양호 (기능 단위로 명확히 구분됨)  
- 협업 방식: 기능 단위 분리 + 주기적 병합  

[개선 제안]
1. 블루프린트 내 반복 로직을 C++로 이전  
2. UObject 의존성 감소 및 모듈 간 참조 정리  
3. Feature 브랜치 병합 주기 단축  
4. GameplayTags 및 DataAsset 표준화  

🧩 추가 지시

모든 분석은 커밋 이력, 파일 구조, 브랜치 이름 등 객관적 근거에 기반해야 한다.

겉보기 요약이 아닌, 개발자의 의도와 기술적 전략까지 추론하라.

불필요한 코드 설명은 생략하고, 시스템 설계·구조·개선 방향 중심으로 서술하라.

결과는 체계적이고 읽기 쉬운 보고서 형태로 제시하라.
